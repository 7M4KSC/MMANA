○数式
~~~~~~
　ＭＭＡＮＡで値を入力する際、ほとんどのケースで簡単な数式を使用することができます。数式で使用できる演算子は「+, -, *, /」の４つだけですが演算の優先順位は考慮されます。また()記号でその順位を変更する事もできます。

	5*2+1		→	11
	1+5*2		→	11
	(1+5)*2		→	12
	5*(2+1)		→	15
	20/(3+7)	→	2

  定数として以下の特別な記号が使用できます。

  R	設計周波数の波長（単位ｍ）
  I	１インチの長さ（単位ｍ）注：１インチ＝２．５４ｃｍ
  F	１ｆｔの長さ（単位ｍ）	注：１ｆｔ＝１２インチ

［例］	R/4	1/4λの長さ
	5*R/8	5/8λの長さ
	30*F	３０フィート
	15*I	１５インチ

　数式を記述した場合、その計算は常にその時に行われます。ワイヤ定義等に数式そのものを記憶しているわけではありません。


○セグメント、テーパリングについて
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　モーメント法では、ワイヤをセグメントという単位で細かく分割し、その分割した各セグメントに流れる電流を計算しますので、どのように分割するかが計算精度に大きく影響します。

  ダイポールや八木等の１本の直線で構成されるエレメントの場合は、均等に１０等分や２０等分にしても、それほど大きな誤差は出ないとされていますが、例えばループアンテナのようにワイヤに折れ曲がりがある場合は分割幅を小さくしないと誤差が大きくなり精度が悪化します。

  テーパリングというのはそれを少しでも改善しようという方法で、折れ曲がり付近（１つのワイヤの端）を細かく分割し、中央部のほうは荒くしていく方法です（すべてを細かく分割するとセグメントが増えて計算時間が増大します）。
  また折れ曲がりのない１本の直線で構成されているエレメントの場合も、テーパリングを設定する事により少ないセグメント数で計算精度を確保できる場合があります。

　いずれの場合でも、インピーダンスの計算精度を上げるには、セグメントの分割幅を細かくする必要がありますが、あまり短く（0.001λ以下）すると逆に計算が不安定になり、また直径とセグメント長の比が４以上あるような太いセグメントでは誤差が大きくなるとされています。
  以下に代表的なアンテナのＳｅｇとＤＭ１、ＤＭ２の設定値を示します。なおこれはあくまで一般例ですので、アンテナの形状や給電の仕方によっては更に細かく分割する必要がある場合もあります。

アンテナ種別		Ｓｅｇ		ＤＭ１		ＤＭ２
ダイポール・八木系	０または−１	200-400		４０
正方形ループ		−１		200-400		４０
三角形ループ		−１		400-600		４０
長方形ループ		−１		400-600		６０
ヘンテナ		−１		400-600		６０
（Segが０の時はDM1の値は結果に関係しません）

  最適化など計算速度を優先させる必要がある場合は、ＤＭ１やＤＭ２を上記の値より減らして計算を行いますが、その場合はどの程度の偏差があるかあらかじめ調べて、それが許容できるかどうかを確認しておくと良いでしょう。一般にこの偏差は、ゲインや指向性特性では少なく、インピーダンス（特にjX）で大きい傾向にあります。例えばあらかじめｊXの偏差を調べておき、最適化の際は「任意のZ」でｊXにその分の逆オフセットを設定して少ないセグメント数で最適化すると短い時間で良い結果が得られる場合があります。

  計算結果について、その精度を判断する基準はありませんが、セグメントの分割数を増減させてインピーダンスが大きく変化しないかなどを調べてみることにより、ある程度推測する事はできます。また電流分布を確認することも重要です。分布が予想に反した結果になっている場合や、変化が粗いと感じる場合は分割数や分割方法を検討すると良いでしょう。
  一般に２つ以上のワイヤがごく接近しているケースや、短く折れ曲がっているもの等の計算精度は悪くなると思います。
  サンプルにはマッチング部分をワイヤ定義で記述してあるような、かなりクリチカルな例も含まれていますが、それらはその働きを試してみたもので、細かい寸法などは信用しないほうが無難だと思われます。


○割り当てパルスについて
~~~~~~~~~~~~~~~~~~~~~~~~
　ワイヤ定義の順番でパルスが割り当てられます。この際、独立したワイヤの両端にはパルスは割り当てられませんが、他のワイヤが接続されている場合は、後から定義したワイヤのその接続点にパルスが割り当てられます（このパルスは後から定義したワイヤに属する）。
　またグランドありで計算する場合に、ワイヤ端のＺ軸の値が（地上高オフセットを含めて）０になるワイヤには、特別にその位置にもパルスが割り当てられます（バーチカルの場合がこのケースになる）。

  ３本以上のワイヤが一点で接続されている場合、一番最初に定義したワイヤにはその端点位置にパルスが割り当てられないので注意が必要です。例えばＧＰアンテナでは５本のワイヤが１点に接続された状態で定義しますが、一番目のワイヤの端（根元）に給電点を設定しても、給電点は接続点よりも１セグメント分内側の位置になります。これを回避するには給電点を設定するワイヤ（ＧＰの場合は垂直のワイヤ）を２番目以降に記述すると良いでしょう。

           給電点=w1b                         給電点=w3b
               (1)                               (3)
                |                                 |
                |                                 |
                |                                 |
                *                                 |
                |                                 |
      (2)-------+-------(3)             (1)-------*-------(2)
      根元から内側にずれる                 根元に給電できる

  また複数のワイヤが１点で接続されている場合、その接続点に給電点を設定する場合は電流の向きにも注意を払う必要があります。例えばダブルダブレッドやツインデルタループの中心位置に給電する場合は、下図の左側のように４本のワイヤの１点に給電してしまうと、正しい給電方法にはなりません。この場合は下図の右側のように給電用のワイヤを１本余分に定義すると良いでしょう（サンプルの「DblDp.maa」，「TwinD.maa」を参照）。

           給電点=w2b                           給電点=w1c
               (2)                                  (2)
                |			             |
                |↑			             |↑
                |			             |   →
     (1)--------+--------(3)		             +--------(3)
           →   |   ←			         (1)/
                |↑			(4)--------+
                |			      →   |
               (4)			           |↑
					           |
                 			          (5)


○集中定数のＳパラメータについて
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　集中定数のＳパラメータは回路をラプラス変換（S=jw）して一般式表現にすると得ることができます。一般式の分子の係数がＡ０〜Ａｎ、分母の係数がＢ０〜Ｂｎです。例えばＭＭＡＮＡが用意したＬＣタイプでは直列共振回路は表現できないので、

                                    R+LS+(1/CS)     1 + RC･S + CL･S^2
 Z = R+jwL+(1/jwC) → R+SL+(1/SC) = ----------- =  ------------------
                                         1           0 + C･S + 0･S^2

となり、A0=1, A1=RC, A2=CL, B0=0, B1=C, B2=0 とすればＯＫです（または A0=1/C, A1=R, A2=L, B0=0, B1=1, B2=0 でも同じ）。なお単位はすべて R[Ω], C[F], L[H]です。この単位系は値が小さくなりがちなので、指数表記で値を指示すると良いでしょう。例えば 20pF は 20×10^-12 ですので、20e-12 のように記述します。

　サンプルの「MCQM.MAA」は更に複雑な回路を定義した例です。「MCQM.MAA」を開き、ファイルメニューの「文書情報」を選択すると解説が表示されますので参照して下さい。

　Ｓパラメータで表現しても、Ｒ＋ｊＸで表現しても、同じ集中定数なので得られる結果は同じですが、Ｒ＋ｊＸの場合は周波数に応じて値を変えなければいけないという欠点があり、マルチバンドのアンテナを解析する場合はかなり煩雑です。なおＬＣタイプは回路を固定していますので、ＭＭＡＮＡが周波数に応じてＲ＋ｊＸの形式に変換します。


○最適化のヒント
~~~~~~~~~~~~~~~~
  最適化のアルゴリズムは色々と試してみましたが、通常のアンテナ調整の際と同様の方法（各変数を個別に変更し、それぞれの最大値を求めながら、その作業を繰り返す）が最も収束が早く結果も比較的満足できる感じです。
  ただしこの方法は「しらみつぶし法」と比較すると真の意味での最適な解を得られる確立は低くなります。解のピークが１箇所でない場合は、小さなピークで収束してしまう事もあるので、結果が満足できない場合は手動でこれはと思うパラメータを変更して（初期値を変えて）再実行する等の処置が必要です。
　また、変数を指定する順序で結果が変わる場合もあり得ます。最適化の際、１番の変数から順に値を変化させていきますので、ここに最も効果的と思われる変数を配置すると良いかも知れません。

  通常、ゲインを追求するとインピーダンスが下がる傾向にあります。極端に低インピーダンスになると狭帯域になり、またワイヤのロスが無視できなくなります。このようなアンテナでは現実のマッチングや再現性に苦労しますので、あまりお勧めできません。Ｒを高めに最適化するには、目標にｊＸではなくＳＷＲ（基準Ｚの値を調整して）を選ぶと良いでしょう。または「マッチング回路」の「任意のＺ」を目標にしても良いでしょう。

　複数のバンドを設定した場合、１回の変数の変化でそのバンドの数だけ試行を行い、それぞれの評価値を合算します。ただし最適化一覧で表示されるのは一番最初の給電情報についての結果だけです。
　モノバンドアンテナでもバンドエッジなどでの目標の悪化を防ぎたい場合は、その周波数を（同一給電点で）登録すると良いでしょう。ただしバンドが増えるとその分試行回数は増加しますので、収束するまで遠大な時間がかかります。また評価値の合算の部分がチューニング不足ですので、あまり期待した結果は得られないかも知れません。

　八木アンテナに限ると、モーメント法では計算速度に難がありますので、起電力法を使った解析ソフトを使用される事をお勧めします。

  以下に最適化の目標の設定例を示します。

・任意のＺを目標にしたい場合
  ＳＷＲ計算インピーダンスに希望のＲとｊＸを設定し、ＳＷＲを目標にして最適化します。
  または詳細ボタンを押してマッチング回路を「任意のＺ」にし、ＲとｊＸに希望の値を設定し、マッチング回路を目標にして最適化します（ＳＷＲとｊＸは目標に含めてはいけない）。

・任意のＲを目標にしたい場合
  詳細ボタンを押してマッチング回路を「任意のＺ」にし、Ｒに希望の値、ｊＸに「*」を設定し、マッチング回路を目標にして最適化します（ＳＷＲとｊＸは目標に含めてはいけない）。

・任意のｊＸを目標にしたい場合
  詳細ボタンを押してマッチング回路を「任意のＺ」にし、Ｒに「*」、ｊＸに希望の値を設定し、マッチング回路を目標にして最適化します（ＳＷＲとｊＸは目標に含めてはいけない）。

・電圧給電のアンテナの共振
  詳細ボタンを押してマッチング回路を「任意のＺ」にし、Ｒを１００００、ｊＸを０に設定し、マッチング回路を目標にして最適化します。この時、ｊＸを少しだけ目標に加えても良いでしょう。またエンドフェッドの場合はｊＸに「*」を設定すると良いでしょう。

・ビームアンテナ
  Ｆ／Ｂ比の計算角度を、水平１８０度、垂直９０度に設定して、目標のＦ／Ｂ比とＧａｉｎを半々ぐらい、ＳＷＲを希望のＺで計算するようにして少し含めると良い結果が得られます。

・広帯域化
  バンド内の２点または３点の周波数を「バンド設定」で設定します。この時、ｊＸよりはＳＷＲを目標にしたほうがＲの変化が少なくなり良い結果が得られます（ＳＷＲの計算Ｚを希望の値に設定しておく）。

・ブーム長を固定する
  エレメントの「位置」を変数に選びます。この時、一番先頭のエレメントと一番最後のエレメントの「位置」を変数にしてはなりません。
  「全エレメント」ボタンを押した時に、エレメントの「位置」を変数として自動登録させるには、あらかじめ「エレメント参照」ボタンを押して、画面下側の「間隔表記」のチェックを外した状態にしておくと良いでしょう。


※「任意のＺ」のＲまたはｊＸに「*」記号を設定すると、その値は評価されなくなります。


○最適化の「全エレメント」ボタン
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  最適化設定画面で「全エレメント」ボタンを押すと、ＭＭＡＮＡが適していると思う順序と組み合わせで、自動的に以下の変数を登録します。

  ループ	全周、間隔（または位置）
  その他	横幅、縦幅、Ｘ幅、間隔（または位置）

	※間隔と位置の選択は「エレメント参照画面」での「間隔表記」
	  のチェックに従う

  ＭＭＡＮＡは上記の登録の際、エレメントの位置関係を推測して、ラジエータ、リフレクタ、ディレクタ（d1〜dn）の順序になるように変数を登録します。
  この時、同一のＸ軸上に存在する変数の現在値が等しい場合、自動的にそれらに連携が設定されます。また同一エレメントの横幅、縦幅、Ｘ幅のうち２つ以上変数が存在する場合、それらをどのように取り扱うかをユーザに問い合わせます（ダイアログボックスが表示されます）。

  同一のＸ軸上に存在しないエレメント間では、「全エレメント」ボタンの操作では連携関係を設定する事はできませんが、最適化設定画面で右ボタンを押しポップアップメニューを表示して、「エレメントの連携設定」を選択すると、既に登録されているエレメント変数に関して、異なるＸ軸上のエレメント間でも、同一の現在値に関して連携関係を自動的に設定する事ができます。
  例えばサーフェースアンテナのように同一幅、同一間隔のエレメントが複数個並ぶ場合は、「エレメントの連携設定」を実行すると便利かも知れません。

  「全エレメント」ボタンを押すと、可変ピッチはええかげんな値が設定されますので、表内の可変ピッチの部分に数値を入力して適当な値に調整してください。またポップアップメニューの「可変ピッチの一括設定」を実行して、すべての変数の可変ピッチを同一の値に変更することもできます。


○スタック展開
~~~~~~~~~~~~~~
  スタック設定画面の「OK」ボタンでスタックが設定された場合、計算時に元のワイヤ定義から指定された数（水平×垂直）と間隔でスタック展開を行った状態で計算を実行します。元のワイヤ定義そのものは変更されないので、簡単に元の状態や他の状態に変更することができます。

  スタック設定画面の「アンテナ定義を変更」ボタンを押すと、アンテナ定義そのものがスタック化した状態に変更されます。この場合は元の状態に戻すことができず、また最適化の際に「スタックの間隔」変数も使用できなくなります。このボタンは、個々のアンテナの給電位相や電圧配分を個別に調整する必要がある場合に使用します。通常は「OK」ボタンを押して、計算時のみスタック展開にする事をお勧めします。

  いずれの場合も、展開するアンテナの数をＮ＝水平の数×垂直の数とすると、

  パルス数	Ｎ倍
  ワイヤ数	Ｎ倍
  給電点数	Ｎ倍 （個々の電圧は１／Ｎ、すべてのアンテナで同じ位相関係）
  集中定数数	Ｎ倍

になります。スタックを設定した場合、ワイヤ数、給電点数、集中定数数については展開可能かどうかチェックされますが、パルス数のチェックはされません。
（計算画面に「PULSE NUMBER EXCEEDS DIMENSION」というメッセージが出る場合はパルス数が計算可能限界を超えています。この場合オプションメニューの環境設定で最大パルス数を増やす必要があります）

  「垂直の展開方法」は垂直方向にアンテナを複数個展開する際に、どの位置を基準にするかを選択します。これは自由空間では関係しませんが、グランドありで計算する場合に、計算画面で設定する「地上高」が、スタックの中心位置なのか、一番上側または一番下側のアンテナなのかを選択できます。

○文書ファイルの関連付け
~~~~~~~~~~~~~~~~~~~~~~~~
　ＭＭＡＮＡはシステムレジストリを一切触らないので、自動では文書ファイルの関連付けを行いません。Windowsのファイルマネージャから文書ファイルをダブルクリックでオープンしたい場合は、手動でファイルの関連付けを設定してください。ＭＡＡ、ＭＡＢ、ＭＡＯ各ファイルのアイコンは「MMANA.EXE」内に用意してありますので、それを利用すると良いでしょう。


○エレメントの共振周波数を得る
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  ビームアンテナを作成する際、ＧＤＭ（グリッドディップメータ）を利用して、あらかじめ各エレメントの共振周波数を所定の周波数に合わせておくという方法が広く行われてきました。
  パソコン上でアンテナ解析により希望の特性のアンテナを設計した場合でも、あらかじめ各エレメントの共振周波数を求めておけば、製作する際にＧＤＭを利用して確認しながら作業を進めることができ有効かも知れません。

  以下にＭＭＡＮＡで各エレメントの共振周波数を得る方法を示します。方法は２通りあります。

１．最適化による方法
  給電点を希望のエレメントのワイヤに設定し、ｊＸのみを目標にして、周波数を変数にして最適化します。ｊＸが０の周波数が共振周波数ですから、最適化した結果得られた周波数が共振周波数になります。最適化の際の試行ピッチはＧＤＭ測定時の誤差の範囲にしておくと十分でしょう。

２．周波数特性による方法
  給電点を希望のエレメントのワイヤに設定し、バンド内の適当な周波数で計算した後、「周波数特性」画面の中の「共振」ボタンを押します。Ｚ表示グラフ内のｆｏが求める共振周波数になります。

  上記のいずれの場合も、アンテナ定義は、ＧＤＭで測定する状態と合わせておく事が重要です。例えば個々のエレメントを単独に測定する事を想定する場合は、他のエレメントが存在しないそのエレメント単独のアンテナ定義にしなければなりませんし、アンテナが組みあがった状態でそれぞれのエレメントを測定する場合は、それと同じアンテナ全体のアンテナ定義にして給電点をそれぞれのエレメントに変更しながら共振周波数を求めておく必要があります。

  現実にはＧＤＭでの測定時に、エレメントとの結合や、周囲（自身を含めて）の影響を回避することは不可能ですので、それらを織り込んで測定結果の評価を行う必要があると思います。


○バーチカルアンテナの地上高
~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  バーチカルアンテナはグランドに接しているワイヤの端点（Ｚ軸）を０にし、計算画面の地上高も０にしておかないと給電がエンドフェッドになってしまい、インピーダンスが正しく計算できません。
  しかし、例えば団地の屋上に設置したバーチカルアンテナのその高さでの垂直指向特性を調べたいと思う場合は良くあります。このような場合は、リアルグランドを選択し「メディア」設定の「高さ」をマイナスに設定して、指向性計算の際の大地面を下げて計算すれば良いでしょう。


○異なる半径のワイヤ（パイプ）の組み合わせ
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  ＨＦ帯のアルミパイプで構成する八木アンテナでは、構造上の問題から、異なる半径のパイプを複数本使用して１本のエレメントを構成するのが普通です。バージョン1.55以前のＭＭＡＮＡではこのような異なる半径のワイヤ（パイプ）が組み合わさった場合は別々のワイヤとしてそれぞれの半径を定義する必要がありました。
  バージョン1.56以降のＭＭＡＮＡでは、１本のワイヤを定義するだけで、あらかじめ設定された分割パラメータに従って、複数個の異なる半径のワイヤを組み合わせた状態に展開して計算する事ができます（この機能を組み合わせワイヤと呼びます）。

  組み合わせワイヤを定義するには、ワイヤ定義またはエレメント編集などで、そのワイヤのＲ（半径）をマイナスの値に設定します。
  更に編集メニュー内の「組み合わせワイヤ設定」を選択し、その同じマイナスのＲに対応する、そのワイヤの組み合わせ方法、構成するワイヤの個々の長さや半径を設定します。

  １つの組み合わせワイヤのパラメータは以下の項目で構成されます。

R(mm)	対象となるワイヤと同じ値（必ずマイナスの値）
Type	組み合わせの方法（中心基準、始点基準）
L0〜L9	個々のワイヤの長さ
R0〜R9	個々のワイヤの展開時の半径

  ワイヤ定義のＲにマイナスの値が設定されていると、ＭＭＡＮＡは計算時に組み合わせワイヤの一覧表から同じＲの値を検索し、そこに記述されているパラメータに従ってワイヤを組み合わせます。一覧表内に同じＲのデータが複数個ある場合は、先に記述してあるパラメータが有効になります。このＲの値そのものは、一覧表の検索のキーとなるだけですので、一覧表とワイヤ定義が一致していれば、どのような値でも構いません。

  以下に「Type」による組み合わせ方法の違いを図示します。

[Type ⇔ または ⇔ *] 中心位置を基準にして下記のように展開されます。

      L4    L3    L2    L1     L0     L1    L2    L3    L4
･･･  -----+-----+-----+-----+------+-----+------+-----+------ ･･･
      R4    R3    R2    R1     R0     R1    R2    R3    R4

  展開状態
                              ----		   短いワイヤ
                         ---+------+---
                   ---+-----+------+-----+---
               -+-----+-----+------+-----+------+- 長いワイヤ

[Type ⇒ または ⇒ *] 始点を基準にして下記のように展開されます。

     L0    L1    L2    L3    L4
  ------+-----+------+-----+------ ･･･
     R0    R1    R2    R3    R4

  展開状態
  ----				短いワイヤ
  ------+---
  ------+-----+----
  ------+-----+------+----	長いワイヤ

  八木アンテナでは「⇔」または「⇔ *」、バーチカルでは「⇒」または「⇒ *」が適しています。

  例えば直径30mm，25mm，20mmの３つのパイプを、それぞれ２ｍ、１．８ｍ、先端までの長さで組み合わせる場合は、一覧表で

	L0=2		R0=15
	L1=1.8		R1=12.5
	L2=99999.9	R2=10

を設定します。L2に99999.9を設定しているのは、L3以降の定義を無効にし、エレメントの長さに関係なく、先端までL2で定義したパイプが使われるようにするためです（L3に０を設定するとL2は自動的に99999.9が設定されます）。
  この状態で例えばワイヤの長さが１０ｍの場合は以下のように展開されます。

[Type ⇔ または ⇔ *]

	2.2m   1.8m   2m    1.8m   2.2m
       ------+-----+------+-----+-------
        20mm   25mm  30mm   25mm   20mm

[Type ⇒ または ⇒ *]

	2m     1.8m         6.2m
       ------+-----+--------------------
        30mm   25mm       20mm

  ＭＭＡＮＡが組み合わせワイヤをどのように展開したかは、アンテナ形状表示で、組み合わせの切れ目を青の□で表示するようにしてありますので、視覚的に確認することができます。またワイヤが選択された状態でマウス右ボタンによるポップアップメニューを開き「組み合わせワイヤ展開表示」を選択すると、どのようにワイヤを組み合わせたかを数値による一覧表で確認できます。

  組み合わせワイヤを設定する場合、そのワイヤのセグメントの分割にも注意を払う必要があります。セグメントの均等分割を指定していても、強制的に組み合わせ位置でワイヤが分割されてしまうために完全には均等分割になりません。アンテナ形状表示でセグメントの分割状態を確認して、DM2の値を調整する必要がある場合があります（５０〜６０程度に増やす）。
  セグメントテーパリングを指定している場合は、「⇔ *」または「⇒ *」を選択すると、そのワイヤの両端の分割ワイヤのみセグメントテーパリングされるようになります。この場合も途中のワイヤのセグメントは均等に分割されるために、DM2の値を調整する必要が生じる場合があります。
  残念ながらＭＭＡＮＡはあまり賢くないので、組み合わせワイヤと、セグメントテーパリングの両方を同時に指定すると、組み合わせ点でセグメント分割幅の過度のアンバランスが発生する場合があります。必ずアンテナ形状表示でセグメント分割状態を確認することをお勧めします。

※Ver1.57以降のＭＭＡＮＡでは組み合わせワイヤのセグメントの分割について、以前(Ver1.56)よりも改善してありますが完璧ではありません。必ずセグメントの分割状態を確認して下さい。

  組み合わせワイヤのサンプルは

	3el20.maa	4el20.maa	4el20hm.maa	5el20.maa
	6el10.maa	8el6mW.maa	2delta20.maa

等を参照して下さい。


○ワイヤ半径を変更した場合の補正方法
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  ワイヤの半径を変更すると、そのアンテナの特性が大きく変化するのが普通です。特に八木アンテナやクワッドなどのビームアンテナでは、インピーダンス特性のみならず、ゲインやF/B比なども周波数ずれによる顕著な特性の劣化が起こります。
  しかし既に希望の特性を得ているアンテナデータのワイヤ（パイプ）の半径を変更したいことは良くあります。また継ぎパイプによるテーパ状のエレメント（組み合わせワイヤ）に変更したい場合もあります。
  そのような場合は以下の手順により比較的簡単に元の特性を取り戻すことができる場合があります。

１．希望のワイヤ半径に変更した状態で一回計算する（※参照）。
２．「周波数特性」画面の「共振」ボタンを押しｆｏを得る。
３．得られたｆｏをアンテナ定義画面の設計周波数に設定する。
４．計算して特性を確認する。ここで元の特性に近い状態になっていない場
    合はこの方法を断念する。近い状態になっている時は次に進む。
５．編集メニューの「アンテナのサイズ」画面で、「Ｙ軸」と「Ｚ軸」のみに
    チェックを付けて、希望（元の）周波数に換算変換を実行する。
    （Ｘ軸にチェックを付けないのはブーム長を変更しないため）
６．希望（元の）の周波数で計算して特性を確認する。
７．必要ならばこの状態で再度１〜６をくり返す（より近づくはず）。

--------- 参考 -------------
※広帯域設計されている多素子の八木アンテナなどではｊXが０になる周波数が複数個存在する場合があります。「周波数特性」画面の「共振」ボタンで求めるｆｏはそのうちの１つだけですので、必ずしもそれが元の特性に該当するｆｏとは異なる場合があります。そこで以下の点に注意を払い、あらかじめ最初の計算周波数をシフトしておくと、正しいｆｏを見つけられる確立が高まります。

・細いワイヤから太いワイヤに変更する場合は、共振周波数が低くシフトする傾向があります。また太いワイヤから細いワイヤに変更する場合は、共振周波数が高くシフトする傾向があります。（ループ系のアンテナではこのシフト関係が逆転します）。

・一定半径のエレメントをテーパ状のエレメントに変更する場合は、共振周波数が高くシフトする傾向があります。逆にテーパ状エレメントを一定半径のエレメントに変更する場合は共振周波数が低くシフトする傾向があります。
----------------------------

  この一連の手順で大体は元の特性に戻ると思います。不満が残る場合は、この状態から最適化を実行します。
  なおこの方法は、元のアンテナが設計周波数で共振していることを前提にしてありますので、例えばヘアピンマッチなどを想定して、少し容量性になっている場合は、一旦ラジエータの長さ修正して共振状態に戻してから、上記の操作を行い、その後再度ヘアピンマッチを想定した状態にラジエータを調整しなおせば良いでしょう。

  各エレメント単独に上記の操作を行うと、より正確に補正できると思いますが、エレメント個別のアンテナ定義を作成する必要があり、かなりメンドウですので、とりあえずはアンテナ全体で試してみると良いでしょう。


○絶縁ワイヤ（Ｒ＝０）を利用する
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  ＭＭＡＮＡのエレメント編集や、最適化の際の「エレメント」変数では、１つのエレメントに含まれるワイヤはすべて接続されていなければなりません。
  しかし現実には途中で分離しているエレメントを、１つのエレメントとして扱いたい場合があります。
  Ver1.63以降のＭＭＡＮＡではワイヤの半径Ｒに０を設定すると、そのワイヤは計算対象から外れるようになっているために、これを絶縁ワイヤとして定義することにより上記のような場合に対処する事ができます。

  またＭＭＡＮＡの技術上の問題から、エレメント編集（変数）では、そのパラメータが変更された場合に、八木やクワッドなどの左右、上下対象形のエレメントを想定して、ワイヤの端点の変更箇所を決定します。したがって対象になっていないエレメントでは、希望通りの動作にならない場合があります。このような場合でも「絶縁ワイヤ」を余分に定義して見かけを対象形にして対処する事ができます。

  以下に絶縁ワイヤの使用例を示します。

［例１］
    -Y                 0                  Y
    --------------           --------------
  この場合、エレメント編集では２つのエレメントに分離されてしまう。

    -Y                 0                  Y
    --------------===========--------------
  中央に「===」で示す絶縁ワイヤを定義すると、１つのエレメントにでき、エレメント編集の「横幅」を変更した場合は、外側の座標のみが変更されるようになる。


［例２］
    -Y                 0                  Y
    --------------
  この場合、エレメント編集で「横幅」を変更した場合は、このワイヤの両端が変化し、ワイヤの右端位置を固定しておきたい場合に困る。

    -Y                 0                  Y
    --------------=========================
  上記のように「===」で示す絶縁ワイヤを接続した状態で定義しておくと、「横幅」を変更した場合、エレメントの両端が変化するのは同じだが、元々のワイヤの右端は固定された状態にできる。


  絶縁ワイヤについて以下の注意点があります。

・絶縁ワイヤに給電点を設定してはいけません。
・絶縁ワイヤに集中定数を設定してはいけません。

  ＭＭＡＮＡは計算時に上記のチェックを行わないので、適当な他のワイヤに給電点や集中定数のパルス位置が代替されてしまいます。


○エレメント編集の「すべての座標を同一比率で変更」について
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  例えば以下のような形状のアンテナの場合、L1=L2=L3=L4とすると、エレメント編集画面で「端点のみ変更」を選択している場合に「横幅」や「全周」を変更した時は、関連するワイヤの先端の座標しか変更されないために、L1とL4は変化しますが、L2とL3は変化しません。

	+----------+----------+----------+----------+
	|          |          |          |          |
	|          |          |          |          |
	|          |          |          |          |
	+----------+----------+----------+----------+
	     L1         L2         L3         L4

  このようなアンテナの場合、L1=L2=L3=L4の関係を常に保つには、「すべての座標を同一比率で変更」を選択した状態で値を変更してください。

  また最適化の際の変数は以下のようになります。

「端点のみ変更」の状態
	横幅、縦幅、全周

「すべての座標を同一比率で変更」の状態
	横幅（全ての座標）、縦幅（全ての座標）、全周（全ての座標）
